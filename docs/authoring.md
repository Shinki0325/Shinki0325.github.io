# Authoring Guide

## Public content

- Add long-form writing under `src/content/articles/`
- Add short notes under `src/content/notes/`
- Add curated topic pages under `src/content/topics/`

## Private content

- Store unpublished drafts under `src/content/drafts/`
- Store private research notes under `src/content/vault/`

## Browser workflow

1. Open the private source repository in GitHub.
2. Press `.` to open `github.dev`, or use the GitHub web file editor.
3. Create or update the Markdown file for an article, note, or topic.
4. Upload any images into the matching public or private folder.
5. Commit changes to `main`.
6. Wait for the deploy workflow to publish the public site.

## Quick content rules

- Use `articles` for finished long-form writing.
- Use `notes` for short ideas, fragments, and image-backed snippets.
- Use `drafts` for work you may publish later.
- Use `vault` for material that must stay private.
