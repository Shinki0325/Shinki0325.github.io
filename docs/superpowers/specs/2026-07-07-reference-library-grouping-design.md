# 参考资料分组与摘要设计

目标是在不改变现有静态发布方式的前提下，把批量导入的参考资料页整理成更像 wiki 节点的结构。用户确认采用三类资料分组：`回忆、讨论与后见视角`、`作品与人物`、`社会背景`。

本次设计包含四个部分：

1. 为参考资料增加 `librarySection` 字段，作为站内公开资料库的一级分组依据。
2. 为每张资料页增加页首“这份资料在讲什么”，优先使用显式 `intro`，缺失时从 `summary + 分组说明` 自动生成。
3. 将资料页的“关联参考资料”改成“显式互链优先，自动推荐补足”的模式，避免新导入页面继续孤立。
4. 为公开参考资料统一整理更短的中文标题与更明确的中文摘要，保留原始来源标题在 `sourceTitle` 中。

索引页展示规则：

- `topic` 类型单独作为“资料包与专题入口”
- `source` 类型按三类分组展示
- 卡片眉标优先展示资料分组，而不是只显示“来源资料”

内容整理规则：

- 标题优先服务浏览与检索，尽量短、尽量中文
- 摘要必须说明“这份资料主要在讲什么”
- 与视频主线强相关的资料优先补显式互链
- 无需为每张页手填完整 `relatedRefs`，由自动推荐补足同类节点

这样处理后，后续继续导入新资料时，只需要补分组、中文标题和摘要，就能自然接入现有资料库结构。
